09KST Base Time 기준 데이터 활용 시나리오 (연속 예보 vs 구간 이어붙이기)

시각적 차이: 시계열 데이터 흐름 비교 (예: 기온)

방식 A: 연속성 유지

하나의 물리 엔진이 계산한 결과로, 모든 시간대의 기온 변화가 자연스럽고 매끄럽습니다.

방식 B: 업데이트 및 단절

매일 최신 정보를 반영하지만, 서로 다른 예보가 만나는 지점(빨간 점선)에서 데이터가 튀는 현상이 발생합니다.

  • 방식 A (연속): 데이터 분석 모델의 입력값으로 사용할 때 좋습니다. 물리적 인과관계가 깨지지 않아 딥러닝 모델 학습 시 노이즈가 적습니다.
  • 방식 B (이어붙이기): 실제 날씨와 가장 가까운 '최종 예보치'를 아카이빙할 때 좋습니다. 실시간 서비스용으로 적합합니다.

⚠️ 그래프의 단절(점프) 주의

방식 B의 그래프에서 빨간 점선 부근을 보세요. 어제는 오늘 밤 기온을 15도로 예보했지만, 오늘 아침 새로 들어온 데이터는 13도로 보고 있습니다. 이 두 조각을 붙이면 밤 12시에 기온이 2도 순간이동 하는 현상이 발생하며, 이를 데이터 불연속성이라고 합니다.
